# EMW UAT & Defect Tracking Dashboard
## EmpowerMe Wellness | Consent Workflow & RCM Transformation

[cite_start]This interactive dashboard tracks the User Acceptance Testing (UAT) phase for the EmpowerMe Wellness system transformation[cite: 3, 4].

### 🚀 Quick Start
1. [cite_start]**Open the Dashboard:** Open `EMW_UAT_Dashboard.html` in Chrome, Edge, or Firefox[cite: 8].
2. [cite_start]**No Setup Required:** The tool runs entirely in your browser with no login or internet required[cite: 9, 46].
3. [cite_start]**Select Your Role:** Use the dropdown in the top-right corner to set your permissions (Tester, Lead, or Stakeholder)[cite: 10, 11, 12].

### 📊 Project Timeline
* [cite_start]**UAT Phase:** May 5–23, 2026 [cite: 6]
* [cite_start]**Go/No-Go Decision:** May 30, 2026 [cite: 6]

### 📝 Testing Workflow
1. [cite_start]**Navigate:** Select your process flow from the sidebar (e.g., "New Consent")[cite: 20, 21, 27].
2. [cite_start]**Execute:** Click a test case to open the Detail Panel and follow the steps against the Crosby/Alinea system[cite: 27].
3. [cite_start]**Log Result:** * **PASS:** Mark as Pass to auto-timestamp[cite: 27].
   * [cite_start]**FAIL:** Mark as Fail to auto-generate a defect (DEF-XXX)[cite: 27, 31].
   * [cite_start]**BLOCKED:** Mark as Blocked if a dependency prevents testing[cite: 27].

### 🐛 Defect Lifecycle
[cite_start]Defects follow a strict path: **Fail** → **Assigned** → **Fixed** → **Retest** → **Pending Approval** → **Closed**[cite: 29, 30, 31]. [cite_start]All final closures require sign-off from Test Leads (Elyse Sallman or Katy)[cite: 31, 33, 49].
